# 2026-07-03 — Smoke harness and stock layout clarity

Added a headless smoke-test harness and improved the stock movement helpers/layout visibility.

Changed:

- Current version is now `0.1.22-smoke-and-stock-layout`.
- Added `npm run smoke` via `prototype/package.json`.
- Added `prototype/scripts/smoke.js` to run a reducer/system-level loop:
  - opening order
  - collection
  - location selection
  - autoload
  - autodisplay
  - trading checkpoint completion
  - end day
  - markdown export check
- Improved autoload balancing.
  - Previous behaviour could pick one of each plant, then fill the rest with one dominant line.
  - New behaviour is round-robin by plant line: oldest batch of each line, then second-oldest of each line, and so on until capacity is full.
- Added `prototype/src/components/SlotSchematic.jsx`.
- Van Loadout now shows a compact van schematic.
- Display Setup now shows a compact 15-slot display schematic split by display area.
- Slot cells show the number of plants left in the tray/batch.
- Potted plant counts are bold, distinguishing them from tray stock.

Still not added:

- browser-driven automated UI tests
- save/load
- staff implementation
- upgrades
- drag-and-drop display
- final art dependency

Notes:

- The smoke harness is a headless state-flow check, not a visual browser test.
- The slot schematic is plain in-game UI, not generated artwork.
